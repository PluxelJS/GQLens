# 架构速览

GQLens 的实现分成六层。

1. `@gqlens/core`
   - `cache.ts`：GraphDataStore，公共 API 使用 `GraphDataAddress`，字符串 key 仅是 store 内部编码。
   - `planner.ts`：把 selection path 合并成 GraphQL operation，自动补 `id` / `__typename`，处理 alias、变量、`.ids`、`.refs`、`$on`。
   - `session.ts`：维护 active selection，按 cache policy 调度 fetch/live subscribe，把响应 normalize 后同步 root/relation/list slot。
   - `invalidation.ts`：统一处理 entity invalidation 和 selector target invalidation。
   - `mutation.ts`：统一执行 mutation descriptor / callback，处理 optimistic、invalidation、normalize 和 rollback。

2. `@gqlens/core/codegen`
   - 运行 generated accessor 的小 runtime。
   - accessor node 是非枚举对象；scalar getter 记录 demand 并读 signal；entity relation 只推进 path；普通列表暴露 `.ids`，抽象列表暴露 `.refs`。
   - `defineSelection()` / `defineInvalidation()` 使用同一套 accessor graph 做纯 path collection。

3. `@gqlens/codegen`
   - `accessor.ts` 用 `code-block-writer` 根据 GraphQL schema 生成 typed node、hooks、metadata、mutation descriptors。
   - object/interface/union 都会生成 node。抽象类型通过 `$on.<TypeCondition>` 表达 inline fragment。
   - 生成文件按 typed nodes、schema metadata、runtime entrypoints、selector builders、mutation descriptors 分段，方便快速阅读。
   - `normalizer.ts` 生成 normalizer metadata 和 typed invalidation union。

4. `@gqlens/vite`
   - Vite plugin 读取 typed `defineGQLensEntry()` default export。
   - dev 用 `server.ssrLoadModule()` 重新加载 schema/handler，build 用 Node native import 读取同一 entry。
   - 对 SDL 做内存 diff，只在类型系统变化时调用 `generateFiles()`。
   - 对 generated 文件做 content-diff 写盘，并可选挂载 `/graphql` dev middleware。

5. 框架适配器
   - React：`packages/react/src/index.tsx`
   - Solid：`packages/solid/src/index.ts`
   - 适配器负责把 render 中的 demand 替换成 session active selection，并监听 signal 触发重渲染。
   - mutation 复用 core runner；适配器只提供 store、fetcher 和成功后 refetch active session 的策略。

6. 传输
   - `transport.ts` 只提供默认 HTTP/live helper。
   - 真正协议、鉴权、重连仍属于应用边界。

核心心智：

- `undefined` 是 missing / 分支不适用，`null` 是服务端返回 null。
- stale 不会清值；读取旧值，同时 active demand 可触发 refetch。
- 列表不是数组数据。普通 entity list 只读 `.ids`，interface/union list 只读 `.refs`。
- 行字段必须重新进入 root/entity accessor 读取。
- `$on` 不是动态探测；可用分支来自 schema metadata。

常用验证命令：

```bash
npm run verify
```

当前全量验证包含 lint、format、typecheck、build、test。
