## Codegen 当前产物

`generateFiles()` 当前输出：

- `types.ts`：GraphQL Code Generator 的 TypeScript 类型。
- `normalizer.ts`：normalizer metadata。
- `invalidation.ts`：导出 `Invalidation = GraphDataInvalidation`，和 runtime target 模型保持同名语义。
- `accessor.ts`：typed accessor、hooks/create functions、selector builders、schema metadata、mutation descriptors。

其中真正驱动 runtime 行为的是 `accessor.ts` 内的 `schemaMeta` 和 generated hook/create function。

`accessor.ts` 由 `code-block-writer` 生成。生成文件固定分为 typed nodes、schema metadata、runtime entrypoints、selector builders、mutation descriptors，metadata 容器按类型/字段展开，方便人和 LLM 快速定位 runtime 所需信息。

## 暂不实现 / 不应进入 Accessor API

- dynamic field API。
- list array API，例如 `.map()`、`[0]`、`.item()`。
- field-level directive / fetch policy chain。
- user-facing alias API。
- runtime `q.invalidate()`。
- accessor node enumeration / spreading as data object。

## 后续设计点

- `normalizerEntries` 是否接入 runtime，需要等 codegen 设计确定后再做。
- Entity / Value Object 分类、embedded address 和 metadata-driven normalize 规则见 [服务端 Schema 设计指南](../服务端-Schema设计指南.md)。
- session 仍可继续轻量抽纯 helper，但不应合并成继承式大抽象。
