# 06. 框架适配

核心对框架只暴露四个能力，适配层仅负责接入各框架的响应式模型：

| 能力               | 说明                                   |
| ------------------ | -------------------------------------- |
| read field         | 执行字段 accessor，返回当前值          |
| track reader       | 将 reader 绑定到 field signal          |
| notify reader      | field signal 变更后通知该 reader       |
| cleanup selection  | 组件卸载时清除其 active selection      |

## React

```tsx
function UserName({ id }: { id: string }) {
  const q = useQuery()
  const name = useField(() => q.user({ id }).name())
  return <span>{name}</span>
}
```

React 无法天然做 field 级 DOM 更新。`useField` 通过 `useSyncExternalStore` 桥接 alien signal，订阅到最小组件边界触发重渲染。

### 订阅粒度原则

```
列表组件   → 订阅 ID 列表
行组件     → 订阅单行字段
字段组件   → 订阅单字段
```

推荐的拆分结构：

```tsx
function TodoList() {
  const q = useQuery()
  const ids = useField(() => q.todos().ids())
  return ids.map(id => <TodoRow key={id} id={id} />)
}

function TodoRow({ id }: { id: string }) {
  return (
    <>
      <TodoTitle id={id} />
      <TodoDone  id={id} />
    </>
  )
}

function TodoTitle({ id }: { id: string }) {
  const q = useQuery()
  const title = useField(() => q.todo({ id }).title())
  return <span>{title}</span>
}
```

## Solid

```tsx
function UserName(props: { id: string }) {
  const q = useQuery()
  return <span>{q.user({ id: props.id }).name()}</span>
}
```

Solid 的 fine-grained runtime 天然追踪 signal 访问，无需包装。适配层只需处理 query scope、selection batching、resource/suspense、cleanup。

## API 对照

| 能力       | React                                   | Solid                           |
| ---------- | --------------------------------------- | ------------------------------- |
| 查询       | `useQuery()`                            | `createQuery()`                 |
| 实时查询   | `useLiveQuery()`                        | `createLiveQuery()`             |
| 字段读取   | `useField(() => q.user({ id }).name())` | `q.user({ id }).name()`         |
| Mutation   | `useMutation(api.user.rename)`          | `createMutation(api.user.rename)` |
